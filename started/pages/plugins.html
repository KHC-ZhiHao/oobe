插件

Plugin

Plugin是實驗性功能，未來隨時會有變動

Plugin顧名思義就是擴展oobe沒有的功能。

let oobe = new Oobe()

class MyFirstPlugin {
    constructor(oobe) {
        // do something...
    }
}

oobe.plugin(MyFirstPlugin)

Loader

Loader是一個官方擴展套件，若你使用npm安裝oobe，則能夠使用以下方法使用Loader:

同時Loader也是一個plugin範本，示範如何藉由監聽生命週期來賦予sprite發送請求的能力。

import Loader from 'oobe/plugins/loader'
let oobe = new Oobe()
oobe.plugin(Loader)

管理自己的狀態

由於經常發現sprite常常需要透過自身的標示符去請求自身的資料，但sprite並不具備請求狀態管理的能力，因此常使用狀態管理工具來協助處理，但寫到後期確實無力，所以可以使用Loader讓sprite來管理一些自身的狀態。

在vue中實現

import Vue from 'vue'
import Oobe from 'oobe'
import Loader from 'oobe/plugins/loader'

let oobe = new Oobe()

oobe.plugin(Loader)

let person = {
    body() {
        return {
            name: null,
            age: null
        }
    }
}

let loader = {
    sprite: {
        // loader前三個參數是固定的，之後可以由控制者添加
        age(sprite, done, error, ...args) {
            axios.get('./age', {
                params: {
                    name: sprite.name
                }
            }).then(result => {
                sprite.age = result.data.age
                done()
            }, err => {
                error(err)
            })
        }
    },
    collection: {
        age(collection, done, error, ...args) {
            // 這是collection的方法
        }
    }
}

oobe.join('company', { sprites: { person } })
oobe.loader.set('company', 'person', loader)

Vue.prototype.$oobe = oobe

<template>
    <div>
        <button @click.stop="fetchAge()" v-if="person.$loader.age.called === false">fetch age</button>
        <div v-if="person.$loader.age.error">{{ person.$loader.age.error }}</div>
        <div v-else-if="person.$loader.age.done">{{ person.age }}</div>
        <div v-else-if="person.$loader.age.loading">Loading...</div>
    </div>
</template>

<script>
export default {
    data() {
        return {
            person: this.$oobe.make('company', 'person').$born({
                name: '小明'
            })
        }
    },
    methods: {
        fetchAge() {
            this.person.$loader.age.start()
                .then(() => {
                    // loader start自身就會回傳promise，並依照done or error決定then還是catch。
                })

            // collection.loader.age.start() 這是基於collection的實現模型
        }
    }
}
</script>

Event

plugin並不能自行發送event，你可以藉由sprite(或collection)發送事件。

以Loader為例：

person.$loader.age.start()
person.$on('#loader.success', (context, name) => {
    console.log(`${name} loader success`)
})

event name並沒有硬性規定，但仍希望由plugin發送的事件參照此格式：#pluginName.eventName
